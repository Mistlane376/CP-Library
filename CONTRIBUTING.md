# 贡献指南

感谢你考虑为 CP-Library 做贡献！🎉

## 🤝 如何贡献

### 报告问题

如果你发现了问题或有改进建议，欢迎提交 [Issue](https://github.com/mistlane376/CP-Library/issues)：

1. 使用清晰的标题描述问题
2. 详细说明问题的具体情况
3. 如果可能，提供复现步骤或示例

### 提交代码

1. **Fork 本仓库**
   ```bash
   # 点击页面右上角的 Fork 按钮
   ```

2. **克隆你的 Fork**
   ```bash
   git clone https://github.com/你的用户名/CP-Library.git
   cd CP-Library
   ```

3. **创建新分支**
   ```bash
   git checkout -b feature/新功能名称
   ```

4. **进行更改**
   - 在相应的分类文件夹中添加或修改模板
   - 确保代码格式清晰，添加必要的注释
   - 如果有必要，在 README 中添加介绍

5. **提交更改**
   ```bash
   git add .
   git commit -m "添加: 新模板名称"
   ```

6. **推送到 GitHub**
   ```bash
   git push origin feature/新功能名称
   ```

7. **创建 Pull Request**
   - 访问原仓库，点击 "New Pull Request"
   - 填写 PR 描述，说明你做了什么

## 📝 贡献规范

### 文件命名

- 使用有意义的文件名
- 例如：`快速幂与逆元.md`，而不是 `模板1.md`

### 内容格式

每个模板文件应包含：

1. **标题**：清晰描述模板名称
2. **代码块**：使用 ```cpp 包裹代码
3. **说明**：算法的原理、适用场景、复杂度等
4. **示例**：如果适用，提供使用示例

### 代码规范

- 使用 C++17 标准
- 变量命名清晰明了
- 添加必要的注释
- 保持代码简洁高效

### 提交信息规范

- `添加: xxx模板`
- `更新: xxx算法的说明`
- `修复: xxx文档中的错误`

## 📚 推荐的贡献内容

- [ ] 补充缺失的常用算法模板
